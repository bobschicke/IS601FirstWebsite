Git commands learned and used:


			

				fork – a separate full repository that requires no permissions from the origin. It can remain a separate repository, or thru a pull request can  be merged back to the origin.

				branch – a working copy of the master at the state of the branch. Used for collaboration.

				clone – pulling down an online repository file to your machine so you can begin to work on it. Only done once.

				git init – creates a repository in the current folder

				git status – what's the status of the current folder (repository)

				git log – show commit history

				git add  - adds a file with changes to the commit que.


				To setup credentials associated with the commit (use global for all repositories)

				git config –global user.email “rws7696@njit.edu”

				git config – global user.name “bobschicke”

				git config –global credential.helper cache – will cache the next set of credentials that you type in.


				git commit -m “commit message” - commit changes locally and include a message

					-a = all changes to all files

				git checkout -b UpdatePage – create a new branch and name it UpdatePage

				git branch -a – list all branches in local and remote repositories

				git checkout master – switch to “master” branch

				git merge  - merges changes from different branch into current branch


				git remote add origin https://github.com/bobschicke/IS601FirstWebsite.git – adds remote repository location (links) to the name given (in this case origin is the name given). This is where Fetches and Pushes come from/go to.

					bobschicke2 origin https://github.com/bobschicke2/IS601Test2.git


				git remote show – shows origin (or whatever the name you gave the link)

				git remote show origin – shows origin location


				git push origin master   - upload master branch to origin remote repository
